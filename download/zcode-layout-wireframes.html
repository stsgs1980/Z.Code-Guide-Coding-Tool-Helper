Zeitgeist Grid

Pattern 15

Стиль Linear/Vercel/Stripe: контент в центральной золотой колонке, боковые зоны для "воздуха". Идеально для премиум-документации.

.zeitgeist-layout {
  display: grid;
  grid-template-columns: 1fr 1.618fr 1fr; /* phi center */
  gap: 2.5rem; /* Fib-val 5 */
  min-height: 100vh;
}

.content-main { grid-column: 2; }
.hero-full { grid-column: 1 / -1; }

/* Responsive */
@media (max-width: 768px) {
  .zeitgeist-layout { grid-template-columns: 1fr; }
}

Плюсы

  • Премиум-вид как у Vercel/Linear
  • Контент в фокусе, минимум отвлекающих факторов
  • Идеально для лонгридов и документации
  • Простая реализация

Минусы

  • Большое пустое пространство на широких экранах
  • Sidebar вынесен в overlay или скрыт
  • Сложнее навигация между секциями

Pattern 12

Bento-style компоновка с явными зонами: hero-баннер, KPI-карточки, основная документация и боковая панель навигации.

.dashboard-golden {
  display: grid;
  grid-template-columns: 3fr 5fr 2fr; /* Fib proportions */
  grid-template-rows: auto 1fr 1.618fr;
  gap: 1.5rem; /* Fib-val 3 */
  min-height: 100vh;
}

.hero-banner { grid-column: 1 / -1; }
.nav-sidebar { grid-row: 2 / 4; }
.main-content { grid-row: 2 / 4; }

Плюсы

  • Всегда виден sidebar навигации
  • TOC на месте — быстрый переход
  • Bento-стиль — современный тренд
  • Хорошо для dashboard-подобного контента

Минусы

  • Более сложная responsive логика
  • Уже основная колонка контента
  • Требует продуманного TOC

Magazine Layout

Pattern 11

Редакционный стиль: главная "статья" (Hero + QuickStart) занимает доминирующую позицию, второстепенные материалы — по сторонам. Идеально для контентных проектов.

.magazine-layout {
  display: grid;
  grid-template-columns: 5fr 3fr 2fr; /* 5:3:2 Fib */
  grid-template-rows: 8fr 5fr 3fr; /* Same Fib */
  gap: 1.5rem; /* Fib-val 3 */
}

.lead-story { grid-column: 1; grid-row: 1 / 3; }
.feature { grid-column: 2 / 4; grid-row: 1; }
.side-a { grid-column: 2; grid-row: 2; }
.side-b { grid-column: 3; grid-row: 2 / 4; }

Плюсы

  • Редакционный вид — как журнал
  • Визуальная иерархия контента
  • Интересная компоновка секций
  • Отлично для лендингов

Минусы

  • Навигация скрыта в overlay
  • Сложнее реализовать scroll
  • Не типично для документации
  • Требует переработки контента

Рекомендация

Для текущего проекта ZCode Guide рекомендуется Вариант 2 (Dashboard Golden):

  • Сохраняет видимый sidebar навигации — ключевой элемент документации
  • Добавляет TOC sidebar справа — улучшает навигацию внутри разделов
  • Bento-стиль современен и трендов
  • Хорошо сочетается с существующими компонентами (Hero, Stats, PromoCards)
  • Минимальные изменения в структуре контента
